## Display labels (D-11)

The `source` field arrives as `'DXNEWS' | 'DX-WORLD' | 'NG3K'`. The label shown to users is:

```javascript
const LABEL_PREFIX = '📰 ';
// Display = LABEL_PREFIX + item.source (already the right casing — just prefix it)
// Examples: '📰 DXNEWS', '📰 DX-WORLD', '📰 NG3K'
```

## Dwell behavior (D-11, Claude's discretion per CONTEXT)

Recommendation from RESEARCH Pitfall 5: minimum 5-second dwell so the label doesn't flicker when adjacent items are from different sources. Implementation: drive `currentSourceIndex` from a `setInterval` paced at `animDuration / news.length` (each item gets equal scroll time). When the dwell would change the label more often than every 5s, clamp to 5s minimum. This is simpler than IntersectionObserver and accurate enough — the visual position of items is uniform with linear keyframe animation.

## Hover-pause (D-13, Claude's discretion per CONTEXT)

Two valid approaches per RESEARCH "Pitfall 6":

- **CSS-only `:hover { animation-play-state: paused }`** — most reliable; survives layout reflows
- **React state via onMouseEnter/onMouseLeave** — testable but can desync on reflow

Use **CSS-only for the hover behavior** (reliability) but **track a `hovered` boolean in React state in PARALLEL** so component tests can assert hover state without relying on JSDOM CSS computation. The CSS rule drives actual animation; the state is for testability.

## Click-navigate (D-13)

Replaces `onClick={() => setPaused(!paused)}` at the current `DXNewsTicker.jsx:218`. New behavior: each item span becomes (or wraps) an anchor with `href={item.url} target="_blank" rel="noopener noreferrer"`. The whole scrolling container's onClick handler is REMOVED. Click target is per-item, so clicking on a source label (separate element) doesn't trigger an item open.

## Empty-state hide (D-07)

The CURRENT line `if (!visible || loading || news.length === 0) return null;` already implements D-07 — the merged feed comes back empty when no source has fresh items, so the existing guard works. Don't regress it. Add a test that passes through fetch returning `{ items: [] }` and asserts the component renders nothing.
